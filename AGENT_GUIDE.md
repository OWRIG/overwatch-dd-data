# Agent Guide

本文档给加载本仓库的 AI agent 阅读，用来约束它在协助用户导出、分析网易 DD / 守望先锋数据时的默认行为。它不是面向普通用户的教程，而是 agent 的操作边界。

## 定位

- 将本仓库视为**个人私有 Skill**，不是官方 SDK、公共爬虫、托管服务、API 中转或可复用的数据产品。
- `datamsapi` 不是公开承诺的开发者 API。不要把它描述成网易、暴雪、战网或相关权利方授权、稳定支持的接口。
- 只协助处理用户本人账号、用户本机环境、非商业、本地分析场景。
- 用户询问法律或账号风险时，可以解释实际风险边界，但不要给出正式法律意见。

## 执行前检查

在执行抓取、拉取、分析或 git 操作前，agent 必须：

1. 先阅读 `README.md`，尤其是合规、隐私、速率控制和免责声明部分。
2. 确认用户是在处理自己已登录的网易 DD 账号数据。
3. 默认选择能满足目标的最小数据范围。
4. 在 staging 或 commit 前检查 `git status --short`。
5. 不得暂存、提交、展示或上传 `scripts/creds.json`、`data.json`、含原始玩家标识的报告、完整 token 或其他私人凭据。

## 默认行为

- 默认只导出 match list，不加 `--detail`。
- 只有用户明确需要逐场队友/敌方数据时，才使用 `--detail`，并先提醒它可能包含其他玩家昵称、ID 和单局表现。
- 保持脚本单线程顺序请求，使用既有 delay 控制。不要降低 `MIN_DELAY`，不要移除限速逻辑。
- 多赛季或大量详情拉取时，建议分批执行并增加间隔。
- 判断胜负必须使用 `matchRet`，不要用 `teamScore > opponentScore` 推断胜负。

## 抓取流程边界

- 必须由用户本人操作网易 DD。不要自动化登录，也不要尝试获取他人凭据。
- `setup_capture.ps1` 会临时启用本地 HTTPS 调试代理，并为当前用户导入本地 CA 证书。
- 抓取结束后，提醒用户运行 `.\setup_capture.ps1 -Cleanup`。
- 如果工作中断且抓取环境可能仍开启，优先提醒用户清理代理和证书。
- 不得打印、总结、粘贴、上传或提交 `creds.json` 里的完整 token。

## 拉取与分析流程

- `scripts/ow_pull.py` 只能使用用户本人的 `token` 和 `roleId`。
- 不要要求用户在聊天里发送 token。优先使用用户本机生成的 `scripts/creds.json`。
- 常规导出优先使用：

```powershell
python scripts/ow_pull.py --seasons 21,22,23 --out data.json
```

- 只有用户明确接受额外隐私暴露时，才使用：

```powershell
python scripts/ow_pull.py --seasons 21,22,23 --detail --out data.json
```

- 分析产物默认视为私有。对外发布摘要前，应删除或聚合玩家昵称、`bnetId`、角色标识和逐场明细。

## 禁止事项

- 不帮助收集、购买、交换或使用他人的 `token`、`roleId`、`bnetId` 或网络流量。
- 不构建或建议构建公共面板、托管 API、多用户服务、高频监控、批量下载器或数据交易用途。
- 不绕过登录、鉴权、加密、风控、验证码、限流或服务端限制。
- 不提高并发，不把脚本改造成多线程/协程爬取器。
- 不在新的公开材料里扩散更多接口细节、私人凭据、原始对战数据或能识别玩家的截图。

## Git 边界

- `creds.json`、`data.json`、`report.txt`、`hero_stats.csv`、`map_stats.csv` 和日志应保持在 git 之外。
- 提交前必须检查 staged 文件，只包含用户期望的源码或文档变更。
- 如果用户准备公开仓库，建议保持私有，或先删除私人数据、直接操作细节，以及任何鼓励批量复用的修改。
